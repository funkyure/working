# ESG 工作戰情中心 — 部署說明

## 快速啟動（5 分鐘）

### 步驟一：建立 Google Sheets

1. 前往 [Google Sheets](https://sheets.google.com) → 新增試算表
2. 將工作表名稱改為 `ESG_Work`（底部標籤右鍵 → 重新命名）
3. 第一列輸入以下欄位標題（A到N欄）：

```
項目編號 | 項目名稱 | 類別 | 子類別 | 狀態 | 優先級 | 開始日期 | 到期日 | 完成度 | 負責人 | 關聯單位 | 文件連結 | 備註 | 最後更新
```

4. 從下方「範例資料」複製內容貼入第二列開始

---

### 步驟二：發布 Google Sheets 為 CSV

1. 功能表 → **檔案** → **共用** → **發布到網路**
2. 左側選「ESG_Work」工作表
3. 右側選「**逗號分隔值 (.csv)**」
4. 按「**發布**」→ 確認

發布後會取得類似這樣的 URL（複製備用，但我們不需要它）：
```
https://docs.google.com/spreadsheets/d/XXXXXXXXXXXXXXX/pub?gid=...&single=true&output=csv
```

---

### 步驟三：取得 Spreadsheet ID

從 Google Sheets 網址列複製 ID：
```
https://docs.google.com/spreadsheets/d/ [這段就是ID] /edit#gid=0
```
例如：`1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms`

---

### 步驟四：部署到 GitHub Pages

**方法 A：直接上傳**
1. 前往 [github.com](https://github.com) → 您的 repository（或新建一個）
2. 上傳 `index.html`
3. Settings → Pages → Source 選 `main` branch → Save
4. 等約 1 分鐘，網址會是：`https://[你的帳號].github.io/[repo名稱]/`

**方法 B：Working Copy（iPhone）**
1. 在 Working Copy 中開啟您的 repo
2. 加入 `index.html`
3. Commit → Push

---

### 步驟五：連結資料

1. 開啟儀表板網頁
2. 點右上角「**⚙️ 設定**」
3. 貼上您的 Spreadsheet ID
4. 按「**儲存並重新載入**」

✅ 完成！之後只要更新 Google Sheets，儀表板開啟時自動拉取最新資料。

## 欄位填寫說明

| 欄位 | 格式 | 範例 | 說明 |
|------|------|------|------|
| 項目編號 | ESG-YYYY-NNN | ESG-2025-001 | 流水號，自行編排 |
| 項目名稱 | 文字 | GRI準則研析 | 簡短描述 |
| 類別 | 固定選項 | 查資料 | 查資料/翻譯/信件/追蹤/會議/報告 |
| 子類別 | 文字 | TCFD | 如 GRI/TCFD/綠色金融/公平待客 |
| 狀態 | 固定選項 | 🟡進行中 | 見下方說明 |
| 優先級 | P0/P1/P2 | P1 | P0最高，P2最低 |
| 開始日期 | YYYY-MM-DD | 2025-05-01 | |
| 到期日 | YYYY-MM-DD | 2025-06-30 | |
| 完成度 | 0-100 | 60 | 不加 % 符號 |
| 負責人 | 文字 | 我 | 我/同事/外部顧問 |
| 關聯單位 | 文字 | 金管會 | 可多個，用/分隔 |
| 文件連結 | URL | https://... | Google Drive 或空白 |
| 備註 | 文字 | 最新進度說明 | |
| 最後更新 | YYYY-MM-DD | 2025-05-17 | 手動更新 |

**狀態選項：**
- `🔴緊急` — P0 或逾期，立即處理
- `🟡進行中` — 正在處理中
- `🟢待處理` — 已指派，尚未開始
- `🔵排程中` — 有明確開始日期但未到
- `⚪️已完成` — 完成（近30天仍顯示）

## 常見問題

**Q: 更新 Sheets 後儀表板沒有更新？**  
A: 儀表板每 5 分鐘自動刷新一次，或按 F5 手動刷新。Google Sheets 發布後有約 1-2 分鐘的緩存延遲。

**Q: 如何新增自訂類別？**  
A: 直接在 Sheets 的「類別」欄填入新值即可，儀表板會自動顯示。篩選列的類別按鈕是固定的，可以聯繫開發者修改 HTML。

**Q: 如何在手機上使用？**  
A: 直接用手機瀏覽器開啟 GitHub Pages 網址，儀表板已支援響應式設計。建議加入主畫面捷徑。

**Q: 可以連結多個 Sheets？**  
A: 目前版本只支援單一工作表。如需擴充，可在 `index.html` 的 `CONFIG` 區塊新增第二個 SHEET_ID。
